## ⚔️ 核心技术迭代：v2.0 泥潭翻车 → v3.0 阈值挽救

### 🎯 v2.0 真实重叠修罗场（默认阈值 0.5）
我们主动打破一刀切的简单规则，让正常与攻击流量的发送字节数（`src_bytes`）差距缩减至极其致命的 **8.61 字节**。在默认生死线下，模型由于“好胜心太强”去死磕纯随机噪声，导致严重过拟合，黑客漏报率高达 **90% 以上**，F1-Score 惨淡跌落至 **0.13 ~ 0.15** 区间。

### 🛡️ v3.0 验证集穷举自救
为了不污染期末考卷（测试集），我们在独立的 **验证集** 上穷举遍历了 99 个分类阈值，通过移动生死线完成极限挽救：
* **LightGBM (最优阈值 0.17)**: F1-Score 被生生从 0.1482 翻倍拔高至 **0.3545**，黑客捕获率（Recall）暴力飙升 **6.2 倍**（达到 55.30%）。
* **RandomForest (最优阈值 0.15)**: 强行换回了全场最高的黑客拦截率（**60.75%**），用 941 次误报的运营成本，换取了校园网防线的绝对安全。
